Algumas perguntas você só faz depois que a ferramenta já virou parte do seu fluxo.

Hoje foram oito conversas sobre Claude Code — desde o efeito real de instruções no comportamento do modelo até configuração de ambiente, passando por plugins essenciais pra PHP.

A primeira conversa foi sobre system prompts e skills. A pergunta foi direta: adicionar frases como "whenever you are responding, always make sure you consider if any relevant skills you can use" nas configurações realmente faz diferença? A resposta curta: sim, instruções no contexto de sistema influenciam comportamento. A mais longa envolve entender quando isso ajuda e quando vira ruído.

Mais tarde, pesquisa de plugins PHP pra usar dentro do Claude Code. A conversa foi longa — 28 mensagens. PHP ainda é uma linguagem com ecossistema vasto e há bastante configuração pra fazer pra ter um ambiente de desenvolvimento produtivo com IA.

No meio do dia, um erro clássico de setup no Windows: Claude Code requer git-bash e se não estiver no PATH, ele trava. O tipo de problema que paralisa o trabalho por 10 minutos e tem solução de 2 linhas depois que você sabe o que é.

Também explorei o "everything-claude-code" — um repositório com extensões e configurações pra usar dentro do VS Code. Instalação não trivial, mas a pesquisa valeu.

E a conversa sobre "OpenClaw" — pesquisando qual serviço cloud faria mais sentido pra uma ideia de projeto. Ainda no nível de exploração, sem compromisso.

Dia de expansão de ambiente mais do que de produto. Às vezes você precisa de um dia assim.
